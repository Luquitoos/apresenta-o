Containerização

Docker & Orquestração de Serviços

Benefícios da Containerização

Isolamento

Cada componente roda em seu próprio ambiente isolado, evitando conflitos de dependências.

Portabilidade

Execução consistente em qualquer ambiente que suporte Docker (dev, teste, produção).

Escalabilidade

Facilita deployment em produção e escalonamento horizontal de serviços.

Desenvolvimento

Setup simplificado para novos desenvolvedores - um comando para rodar tudo.

Configuração Docker

docker-compose.yaml

# Services Configuration
version: '3.8'
services:
backend:
build: ./backend
ports: ["5000:5000"]
env_file: .env
frontend:
build: ./frontend
ports: ["3000:3000"]
depends_on: [backend]
Backend Dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
EXPOSE 5000
CMD ["npm", "start"]
Frontend Dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["npm", "start"]
